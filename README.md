# X7 AMC Macro Editor

一個給 X7 / OSCAR Editor `.amc` 巨集使用的非官方瀏覽器版編輯器。目標是保留原廠 `.amc` 匯入格式，同時提供比較容易操作的逐列結構化編輯介面。

## 功能

- 匯入、編輯、匯出標準 `.amc`
- 儲存/開啟編輯器專用 `.x7proj` 專案檔
- 支援鍵盤、滑鼠、延遲、移動、註解、跳行、迴圈、按鈕條件、變數條件與變數指定
- 變數 UI 顯示為 `A / B / C / D`，匯出時對應 `varE / varF / varG / varH`
- `按一下` 會以單列顯示，匯出時展開為 `KeyDown / Delay / KeyUp / Delay`
- 滑鼠點擊也會以單列顯示，匯出時展開為按下/延遲/彈起/延遲
- `按一下` 和滑鼠點擊都可以拆解，方便在中間插入其他指令
- 指令列表支援上移、下移、複製、刪除、行號拖曳排序
- 排序、插入、刪除時會嘗試同步修正跳行/迴圈目標行
- 輔助頁籤可批次修正鍵盤/滑鼠按下與彈起後緊接的延遲
- 使用 SVG overlay 顯示迴圈與跳行視覺輔助，展開列時不會切斷流程線
- 內建變數範本，可用記事本測試變數、跳行與迴圈
- 可複製 OSCAR Editor 巨集資料夾路徑，方便貼到檔案總管
- 提供原廠下載頁連結，方便查找 OSCAR Editor 相關資源

## 使用方式

直接用瀏覽器開啟 `index.html`。

建議使用 Chromium 系瀏覽器，例如 Chrome 或 Edge。部分檔案儲存能力會受瀏覽器安全限制影響；如果瀏覽器不支援直接覆蓋儲存，匯出會改用下載檔案。

## 主要檔案

- `index.html`：介面結構
- `styles.css`：CSS 入口，依序載入 `css/` 下的版面、工具、指令列表與響應式樣式
- `app.js`：編輯器狀態、UI 操作、匯入/匯出流程
- `js/`：鍵盤資料、共用工具與變數範本
- `css/`：依介面區域拆分的樣式檔
- `macro-core.js`：`.amc` 解析、序列化、行號與語法分析
- `picker.html` / `picker.js`：獨立取點小視窗
- `docs/`：AMC 格式、專案格式與公開前檢查清單

## 專案檔與匯出

`.amc` 是給原廠軟體匯入使用的標準輸出。

`.x7proj` 是本編輯器專用的專案格式，用來保留較高階的編輯狀態，例如 AMC 包、檔名設定與結構化資料。最終仍可匯出成單一 `.amc`。

## 組合

組合頁籤提供常用組合，可直接插入 AMC 包、變數迴圈、固定迴圈與條件分支骨架。插入後會自動選到動作占位註解，接著按鍵盤、滑鼠或其他指令即可放進骨架內部。

## 行號

指令列表、AMC 包預覽與 Syntax 面板中的原始 Syntax 都會顯示目前的全局語法行號。這些行號只用於檢視，不會寫入匯出的 `.amc`。

## 取點視窗

主畫面可開啟 `picker.html` 小視窗進行座標取點。小視窗會和主視窗同步絕對/相對座標；可連續暫存多個取點、加入備註並刪除單筆，再回主視窗用絕對/相對清單下拉套用。取點後回到主視窗，變更過的座標欄位會短暫反橘提示。

## 快捷鍵

- `A`：切換新增指令面板
- `S`：隱藏/顯示下方鍵盤與滑鼠面板
- `E`：切換點選指令列是否自動展開

在輸入框、下拉選單或文字區中按鍵時，不會觸發這些快捷鍵。

## 注意事項

- `MacroLibrary/` 是本機參考資料夾，不屬於此 repo 內容，已在 `.gitignore` 排除
- 瀏覽器不能繞過 Windows 對 `Program Files` 的寫入限制
- 「巨集資料夾」按鈕只會複製資料夾路徑，不會直接開啟檔案總管
- 若要把匯出的 `.amc` 放進原廠巨集資料夾，建議先匯出到可寫入的位置，再用檔案總管移動或貼上
